# 贡献指南

感谢你对项目管理系统的关注！

## 如何贡献

### 报告问题

- 使用 [GitHub Issues](https://github.com/openclaw/project-management/issues)
- 清晰描述问题
- 如有可能，提出改进建议

### 提交更改

1. Fork 本仓库
2. 创建特性分支
3. 进行修改
4. 如有可能，用真实项目测试
5. 提交 Pull Request

### 文档指南

添加或修改文档时：

1. 遵循 `docs/` 中的现有结构
2. 保持实用和可操作
3. 必要时包含示例
4. 如添加新文件，更新 `data_structure.md`

### 模板指南

添加或修改模板时：

1. 放在 `templates/` 目录
2. 包含清晰的说明
3. 用户特定内容使用占位符
4. 保持简洁专注

## 项目结构

```
docs/           → 详细规则和指南
templates/      → 开箱即用的项目文档
tools/          → 可选工具
```
