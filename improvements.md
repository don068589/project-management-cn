# 待改进清单

> 系统数据文件 — 正常读写，无需审批。
> 收集系统改进建议。审批通过后执行改动，记录到 README.md 变更日志。

## 格式

| 编号 | 问题/想法 | 来源 | 建议改动 | 状态 |
|------|-----------|------|----------|------|
| | _(暂无)_ | | | |

## 状态值

- `待审阅` — 刚提出，等待决策者审阅
- `已批准` — 决策者同意，待执行
- `已执行` — 改动已完成，记录在 README.md 变更日志
- `已否决` — 决策者不同意，记录原因
- `需补充` — 需要更多信息
